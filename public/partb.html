Collision Count Forecasting

Applying Machine Learning and statistical techniques to forecast traffic collisions count for California's most populated counties.

Summary

The target of forecast is daily collision counts for three counties - Los Angeles, San Diego and Orange County. These counties account for more than 40% of Californian population.

The historical collision data is from the Statewide Integrated Traffic Records System (SWITRS), made available in the Kaggle dataset

To build additional features, the following data is also used:

  • Weather data from the Global Historical Climatology Network
  • Holiday data from python-holidays library
  • Daylight data from the Astronomical Applications Department

Techniques

Forecasting are implemented using two techniques: Random Forest regression in SparkML and Facebook Prophet.

Historical data is split into a train set (2010-2018) and test set (2019). Cross validation is applied, with Root Mean Squared Error (RMSE) and correlation used mainly to evaluate different models.

We decided not to include data during COVID-19 period in our analysis because (1) collisions data are only complete until 2020 end, making validation and testing of model with COVID-19 impact difficult and (2) as COVID-19 is a one-time phenomenon and countries are going back to normal period in 2022, a model that incorporate such historical event might not prove very useful.

Evaluation Results

Following is the table of evaluation scores generated by predictions against test data (year 2019).

County Random Forest Regression Facebook Prophet
RMSE Adjusted RMSE r RMSE Adjusted RMSE r
Los Angeles 38.2002 10.06% 0.731813 36.5843 9.63% 0.764763
San Diego 10.8489 12.33% 0.552153 11.1154 12.63% 0.606867
Orange 14.3794 12.18% 0.562426 12.9818 11.00% 0.689165

Overall, it can be concluded that Facebook Prophet performs better than Random Forest Regression, with better scores across all counties.

This can be explained as Prophet is developed to work best with data exhibiting strong seasonal effects and can detect trend changes, while the Random Forest Regression model could not fully capture the seasonal patterns due to limited number (7 days) of lag features.
